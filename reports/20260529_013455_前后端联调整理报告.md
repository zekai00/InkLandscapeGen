# 前后端联调整理报告

更新时间：2026-05-29 01:34:55

## 1. 当前可运行入口

后端入口：

- `system/backend/app/main.py`
- `system/backend/app/routers/generation_router.py`
- `system/backend/app/services/image_generator.py`

前端入口：

- `system/frontend/src/main.js`
- `system/frontend/src/App.vue`

旧版前端入口已归档到 `system/frontend/src/legacy/`；旧版 SD 1.5、LoRA 和旧图片路由代码已归档到 `system/backend/app/legacy_sd15/`。

## 2. 后端生成链路

后端已从旧 SD 1.5/LoRA 生成链路切换为：

```text
请求 /api/v1/generate
  -> generation_router.py
  -> image_generator.generate()
  -> 本地图像生成服务
  -> system/backend/app/static/generated/*.png
  -> 返回图片 URL、base64、耗时、种子、简化场景图
```

默认使用本地图像生成服务。

默认不加载 LoRA。

已在图像生成服务中补充本地 tokenizer 聊天模板，使本地推理管线可以正常执行提示词编码。

## 3. 前端联调链路

前端页面提供：

- 诗词或自然语言输入。
- 默认、水墨、青绿、浅绛四类风格选择。
- 宽度、高度、推理步数、随机种子设置。
- 调用 `/api/v1/generate` 并展示生成图。
- 展示耗时、种子、尺寸和简化场景图。
- 前端默认使用 256x256、1 step，优先保证联调可完成；正式出图时可手动调高尺寸和步数。

生产构建后的默认 API 地址为：

```bash
http://127.0.0.1:8000
```

如需部署到其他机器，构建前设置 `VUE_APP_API_BASE`。

## 4. 运行命令

启动后端：

```bash
cd /root/Workspace/ControllableImageGeneration/system/backend
PYTHONPATH=/root/Workspace/diffusers/src uvicorn app.main:app --host 127.0.0.1 --port 8000
```

默认使用 GPU 直接推理；显存不足时可开启 `IMAGE_CPU_OFFLOAD=1` 回退到 CPU offload。

只调界面、不跑真实模型时：

```bash
cd /root/Workspace/ControllableImageGeneration/system/backend
MOCK_GENERATION=1 PYTHONPATH=/root/Workspace/diffusers/src uvicorn app.main:app --host 127.0.0.1 --port 8000
```

构建并启动前端静态服务：

```bash
cd /root/Workspace/ControllableImageGeneration/system/frontend
npm install
npm run build
python -m http.server 1024 --bind 127.0.0.1 --directory dist
```

访问：

```bash
http://127.0.0.1:1024/
```

## 6. 注意事项

- 真实图像生成首次推理会加载本地生成服务，时间较长；当前默认使用 GPU 直接推理。
- 如果显存不足，可以设置 `IMAGE_CPU_OFFLOAD=1` 回退到 CPU offload。
- Vue 开发服务器在当前环境可能遇到文件监听数量不足导致的 `ENOSPC`，因此当前推荐使用生产构建加静态服务联调。
- `npm run build` 存在 bundle size 警告，主要来自 Element Plus 和现有图片资源，不影响当前联调。
