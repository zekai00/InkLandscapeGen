# 系统工程增强与第六章同步报告

生成时间：2026-05-30 03:02:38

## 一、修改目标

本次修改主要回应答辩教师关于系统章节“工程深度不足”的意见，在不改变论文核心研究主线的前提下，将系统定位从简单界面展示调整为对第三章数据、第四章场景图扩充和第五章图像生成算法的工程化验证平台。

同时，全文表述统一为“三个研究难点、三个创新点”。系统实现保留为主要工作和工程验证内容，不再作为独立创新点表述。

## 二、系统工程增强内容

### 1. 后端任务队列与资源控制

修改文件：

- `/root/Workspace/ControllableImageGeneration/system/backend/app/routers/generation_router.py`

新增或强化内容：

- 增加 `MAX_GENERATION_QUEUE` 环境变量，用于限制排队和运行中的生成任务数量。
- 当队列达到上限时，接口返回 HTTP 429，避免请求无限堆积。
- 增加 `MAX_TASK_RECORDS`，定期裁剪历史任务对象，防止长期运行时内存持续增长。
- `/api/v1/health` 返回当前活跃任务数、队列上限、可用槽位等状态信息，便于前端提示和系统监控。

### 2. 图像文件访问安全

修改文件：

- `/root/Workspace/ControllableImageGeneration/system/backend/app/routers/generation_router.py`

新增或强化内容：

- 对生成图像文件名进行安全检查，只允许访问生成目录中的文件。
- 拒绝包含路径穿越风险的文件名，例如包含上级目录或子目录跳转的路径。
- 删除历史记录或访问生成图像时统一走安全路径解析逻辑。

### 3. 历史记录可靠写入

修改文件：

- `/root/Workspace/ControllableImageGeneration/system/backend/app/routers/generation_router.py`

新增或强化内容：

- 历史记录写入改为临时文件写入后原子替换，降低进程异常中断导致 `history.json` 损坏的风险。
- 增加 `MAX_HISTORY_RECORDS`，保留最近的有效生成记录。

### 4. 部署配置增强

修改文件：

- `/root/Workspace/ControllableImageGeneration/system/backend/app/main.py`
- `/root/Workspace/ControllableImageGeneration/system/backend/README.md`

新增或强化内容：

- CORS 来源改为环境变量 `CORS_ORIGINS` 配置。
- 增加 `CORS_ALLOW_CREDENTIALS`，避免默认开放凭证跨域。
- README 中补充生成图像目录、队列上限、任务保留数量、历史记录保留数量和跨域配置说明。

### 5. 前端状态提示增强

修改文件：

- `/root/Workspace/ControllableImageGeneration/system/frontend/src/App.vue`

新增或强化内容：

- 前端根据后端健康检查接口显示队列容量、活跃任务数量和剩余可提交任务数量。
- 用户在生成前可以看到系统是否繁忙，减少重复提交和无反馈等待。
